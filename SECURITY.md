# Security Policy

## Supported Versions

We recommend always utilizing the latest version on the `main` branch or the live hosted version on GitHub Pages.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |

## Reporting a Vulnerability

As an educational repository focused on Threat Modeling and Application Security, we take security disclosures seriously.

If you discover a security vulnerability in this application or find inaccurate/misleading security guidance within our curriculum:

1. **Do NOT open a public issue.**
2. Email the repository maintainers or use GitHub's **Private Vulnerability Reporting** feature via the **Security** tab.
3. Include detailed steps to reproduce, impact assessment, and any suggested remediations.

### Response Timeline
* **Acknowledgement**: Within 48 hours.
* **Assessment & Fix**: Within 7 business days for critical issues.

Thank you for helping keep our community and learners secure! 🛡️
